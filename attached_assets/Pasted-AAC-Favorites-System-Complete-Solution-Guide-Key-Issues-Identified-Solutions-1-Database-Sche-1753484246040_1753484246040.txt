📊 Database Schema Understanding
Your schedule_user_favorites table structure:
sqlCREATE TABLE schedule_user_favorites (
  id int4 PRIMARY KEY,
  user_id uuid NOT NULL REFERENCES auth.users,
  activity_id text NOT NULL,           -- Stores vocabulary card ID as string
  activity_data jsonb NOT NULL,        -- Stores complete card data
  created_at timestamp NOT NULL
);
What goes in activity_data:

🔄 Migration Script (If Needed)
If you have existing favorites with null activity_data, run this migration:
sql-- First, identify favorites that need migration
SELECT * FROM schedule_user_favorites WHERE activity_data IS NULL;

-- Update them with vocabulary card data
UPDATE schedule_user_favorites 
SET activity_data = (
  SELECT jsonb_build_object(
    'id', svc.id,
    'categoryname_en', svc.categoryname_en,
    'text_en', svc.text_en,
    'text_es', svc.text_es,
    'spoken_word_en', svc.spoken_word_en,
    'spoken_word_es', svc.spoken_word_es,
    'icon_url', svc.icon_url,
    'sort_order', svc.sort_order,
    'created_at', svc.created_at
  )
  FROM schedule_vocabulary_cards svc 
  WHERE svc.id = CAST(schedule_user_favorites.activity_id AS INTEGER)
)
WHERE activity_data IS NULL;
